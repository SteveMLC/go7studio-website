# Go7Studio AI Lab — Visual Retrofit Map

Date: 2026-04-19
Purpose: exact visual treatment plan for the 4 live AI Lab cornerstone posts so implementation can move fast without inventing the visual rhythm from scratch.

## Shared rule
Every AI Lab post should include:
- 1 thesis/hero callout
- 1 key takeaways block
- 1 comparison/decision matrix or structured table
- 1 checklist/action block
- 1 diagram/process visual/code-treatment section
- visual interruption roughly every 3-4 paragraphs

## 1) harness-engineering-matters-more-than-prompts

### Visual inserts
1. **HeroCallout** near top
   - Thesis: prompts matter, but the harness decides whether the work survives real use
2. **KeyTakeaways** after intro
   - 4-5 bullets: task framing, permissions, context, routing, review loops
3. **DecisionMatrix**
   - `Prompt-centric workflow` vs `Harness-centric workflow`
   - rows: repeatability, safety, quality control, scale, human review
4. **DiagramFrame**
   - simple 5-layer workflow: prompt -> tools -> permissions -> context/memory -> review gate
5. **ChecklistBlock** at end
   - "Build your first harness" checklist

## Styling notes
- Keep existing dark premium Go7Studio aesthetic
- Use sharper editorial cards inside post bodies
- Monospace accents for technical labels and filenames
- Strong borders and subtle glow, not soft blob styling
- Code blocks should feel expensive and purposeful
- Tables should be readable and hoverable
