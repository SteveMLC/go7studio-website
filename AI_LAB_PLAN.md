# Go7Studio AI Lab Plan

Date: 2026-04-19
Status: approved direction
Owner: Steve

## Strategic decision

Go7Studio should own the **AI Lab** content lane.

This lane is for:
- AI tool guides
- operator workflows
- harness engineering field notes
- model comparisons
- real build logs from app/game/automation work

Qualora should only receive the **job-specific derivative versions** when a topic maps directly to a learner or career outcome.

## Why this belongs on Go7Studio

### Audience fit
AI Lab content is for:
- builders
- developers
- founders
- AI power users
- automation operators
- potential clients who may hire for apps, games, and digital product work

That aligns directly with Go7Studio's identity as:
- games
- apps
- digital experiences
- transparent builder brand

### Business upside
This lane can support:
- overall Go7Studio trust and brand building
- inbound contract / consulting opportunities
- future low-ticket AI operator products
- credibility for games/apps built with modern AI workflows
- selective downstream traffic into Qualora when a post has a role-training angle

### Brand protection
Qualora has a clearer and narrower wedge around:
- career learning
- AI-for-work guidance
- role-specific training
- healthcare/compliance-adjacent learning content

So generic AI operator content should not be the main Qualora editorial lane.

## Positioning

## Site structure

### Proposed section
`/ai-lab`

### Initial information architecture
- `/ai-lab` — landing page / hub
- `/ai-lab/guides` — tool guides
- `/ai-lab/comparisons` — tool vs tool comparisons
- `/ai-lab/field-notes` — real build logs, lessons, failure writeups

This can start simple as a single hub page plus tagged posts, then split later if content volume justifies it.

## Content pillars

Examples:
- How to use Claude Code without drowning in context

- Where MiniMax fits and where it doesn't

### 2. Comparisons
Examples:

### 3. Field notes
Examples:

### 4. Workflow playbooks
Examples:
- My bug-fix workflow with AI coding agents
- How I review AI-generated code before shipping

### 5. Governance and safety
Examples:
- When not to let the model act
- Human-review rules I won't remove
- Permission models for coding agents
- Why memory design matters

## Content routing rule

### Publish on Go7Studio when the core question is:
- which tool should I use?
- how do I build with this?
- what workflow works best?
- what failed and why?
- how do I structure the harness / review loop?

### Publish on Qualora when the core question is:
- how does this help me in my job?
- how do people in a role use this safely?
- what do I need to learn to apply this at work?
- what training path should I follow?

## Go7Studio → Qualora bridge

Use a light CTA block on relevant AI Lab posts:
- **Builder path:** work with Go7Studio / get the guide / join the email list
- **Learner path:** if you're here to learn this for work, see the related Qualora training

Do not force every AI Lab post to push Qualora. Only add that CTA when it is a clean fit.
